Scrapy Commands

发表于 2018-03-04

Available tool commands

Global commands

1. startproject

scrapy startproject <project_name> [project_dir]

$ scrapy startproject myproject

在 project_dir 文件夹下，创建一个名为 project_name 的爬虫项目。

2. genspider

scrapy genspider [-t template] <spider_name> <domain>

$ scrapy genspider example example.com
$ scrapy genspider -t crawl scrapyorg scrapy.org

在当前项目的 spider 文件夹下创建一个名为 spider_name 的爬虫. domain 用来生成 allowed_domains和 start_urls。

template选项

  • basic(default)
  • crawl
  • csvfeed
  • xmlfeed

3. settings

scrapy setting [options]

$ scrapy settings --get BOT_NAME

查看项目的配置.

4. runspider

scrapy runspider <spider_file.py>

$ scrapy runspider myspider.py

不创建项目，直接运行一个包含在Python文件中的爬虫。

5. shell

scrapy shell [url]

$ scrapy shell http://www.example.com/some/page.html
$ scrapy shell --nolog http://www.example.com/ -c '(response.status, response.url)'

为指定的 url 启动Scrapy shell，如果没有 url， 则打开一个空的 shell。

可选命令：

  • --spider=SPIDER: 绕过爬虫的自动检测，并使用特定爬虫
  • -c code: 检查shell中的代码，打印结果并退出

6. fetch

scrapy fetch <url>

$ scrapy fetch --nolog http://www.example.com/some/page.html

使用 Scrapy downloader 下载指定 url 的内容，并将内容写入标准输出。

可选命令

  • --spider=SPIDER: 绕过爬虫的自动检测，并使用特定爬虫
  • --headers: 输出响应头，而不是响应正文

7. view

scrapy view <url>

$ scrapy view http://www.example.com/some/page.html

在浏览器中打开 url，并用Scrapy检查。

8. version

scrapy version [-v]

打印 Scrapy 版本。 如果与 -v 一起使用，它还会打印 Python，Twisted 和 Platform 信息，对于错误报告很有用。

Project-only commands

1. crawl

scrapy crawl <spider>

运行爬虫。（运行前常用 list 列出并检查爬虫）

2. check

scrapy check [-l] <spider>

检查爬虫。

3. list

scrapy list

列出当前项目下的所有爬虫。

4. edit

scrapy editor <spider>

使用 EDITOR 环境变量或 EDITOR 设置中定义的编辑器编辑 spider。

5. parse

scrapy parse <url> [options]

$ scrapy parse http://www.example.com/ -c parse_item

获取 url 然后使用 parse 方法解析它。若果有 --callback 选项，则使用 --callback 传递的方法解析。

可选命令

  • --spider=SPIDER: 绕过爬虫的自动检测，并使用特定爬虫
  • --a NAME=VALUE: 设置蜘蛛参数（可能会重复）
  • --callback or -c: 使用指定的 spider 作为解析响应的回调函数
  • --pipelines: 通过pipelines处理项目
  • --rules or -r: 使用 CrawlSpider 规则来获取用于解析响应的回调函数
  • --noitems: 不显示爬到的 items
  • --nolinks: 不显示提取的衔接
  • --nocolour: 避免使用 pygments 来着色输出
  • --depth or -d: 递归爬取请求的深度（默认值：1）
  • --verbose or -v: 显示每个深度级别的信息

6. bench

scrapy bench

进行一个快速的基准测试。

Custom project commands

可以通过 COMMANDS_MODULE 的设置来添加自定义项目命令。
